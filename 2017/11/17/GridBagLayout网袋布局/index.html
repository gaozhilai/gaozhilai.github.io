布局管理器GridBagLayout网袋布局

2017-11-17 • 标签: GridbagLayout, swing

定义:

GridBagLayout网袋布局,Java Swing中的一种布局方式,可以写出很复杂的布局,最近几天给我的感受就是,虽然Java Swing很古老,但是只要CardLayout与GridBagLayout相配合,没有什么复杂布局是做不出来的.

使用方法:

将父容器设置为GridBagLayout然后为添加的组件设置GridBagConstraints即可.

GridBagConstraints参数及含义:

GridBagLayout很强大,主要难点在于GridBagConstraints,下面分析GridBagConstraints各个参数的含义.

@gridx,gridy:
组件左上角所在的位置，如上图中左侧的面板在1行0列，则gridy=0,gridx=1。读者请注意这里的行对应的是gridy,列对应的是gridx

@gridwidth,gridheight
组件占据的行数和列数，如最上面的那个面板占了1行2列，则gridwidth=2,gridheight=1

@weightx,weighty
可以简单理解为组件大小变化的增量值，如设置weightx=100，组件会随着单元格而变化，设置weightx=0时，组件大小不会发生变化。当然weightx,weighty也可以设置成其他的值，不过意义不大，就不再详细介绍。

@fill
组件在所处格子(分配区域)内的填充方式
如fill= HORIZONTAL,组件就只在水平方向上填充满单元格，取fill= BOTH则会填满整个格子。

@anchor
组件在所处格子内的对其方式，取anchor=EAST就是指右对齐

@ipadx,ipady
内部填充，是指在组件首选大小的基础上x方向上加上ipadx，y方向上加上ipady,这样做就可以保证组件不会收缩到ipadx,ipady所确定的大小以下，因此我们可以用ipadx,ipady的值来指定组件的大小，而不必指定组件的大小否则会有意想不到的效果

@insets
外部填充，填充的区域是组件与所处格子边框之间的部分,有left,top,right,bottom四个参数，不过当组件的fill=NONE时，指定insects值是无意义的

敲黑板划重点:

GridBagConstraints最基本的用法是先new一个对象然后设置各种参数,然后与对应的组件绑定,以后每个组件无限重复这一过程,繁琐枯燥乏味.
GridBagConstraints进阶用法是只new一个对象然后设置各个属性与组件绑定,下一个组件先重新设置这个对象各个参数然后复用这个对象,这种方法经常有人炫技使用,但是只要是人用这个方法就难免老马失前蹄.极度不推荐这种用法.
在网上有人给出了一个工具类大大简化了这一过程,从此以后即使每个组件都是新new的约束对象代码也不再繁琐,例如: 

this.add(rightResult,new GBCUtil(3,2,1,1).setFill(GBCUtil.BOTH).setWeight(20,100));

一句代码搞定.下面给出工具类.

工具类GBCUtil

import java.awt.*;
public class GBCUtil extends GridBagConstraints{
//初始化左上角位置
public GBCUtil(int gridx, int gridy)
{
    this.gridx = gridx;
    this.gridy = gridy;
}

//初始化左上角位置和所占行数和列数
public GBCUtil(int gridx, int gridy, int gridwidth, int gridheight)
{
    this.gridx = gridx;
    this.gridy = gridy;
    this.gridwidth = gridwidth;
    this.gridheight = gridheight;
}

//对齐方式
public GBCUtil setAnchor(int anchor)
{
    this.anchor = anchor;
    return this;
}

//是否拉伸及拉伸方向
public GBCUtil setFill(int fill)
{
    this.fill = fill;
    return this;
}

//x和y方向上的增量
public GBCUtil setWeight(double weightx, double weighty)
{
    this.weightx = weightx;
    this.weighty = weighty;
    return this;
}

//外部填充
public GBCUtil setInsets(int distance)
{
    this.insets = new Insets(distance, distance, distance, distance);
    return this;
}

//外填充
public GBCUtil setInsets(int top, int left, int bottom, int right)
{
    this.insets = new Insets(top, left, bottom, right);
    return this;
}

//内填充
public GBCUtil setIpad(int ipadx, int ipady)
{
    this.ipadx = ipadx;
    this.ipady = ipady;
    return this;
}
}
