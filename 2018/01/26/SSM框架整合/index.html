单独使用Mybatis

Mybatis最重要最核心的对象就是SqlSessionFactory,用以产生SqlSession,执行各种数据库相关操作.单独配置Mybatis的时候需要在配置文件中配置
实体类别名typeAliases
数据源dataSource(属于环境标签下environments–>environment的内容)
要加载的mapper(属于mappers–>mapper下的内容)
然后在dao层实现类中用如下代码即可获得SqlSessionFactory进行数据库操作
(此处的mybatis单独使用,官方教程建议将SqlSessionFactory封装一个单例工具类)

String resource = "originalmybatisway/mybatis-config.xml";
InputStream inputStream = Resources.getResourceAsStream(resource);
SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
SqlSession sqlSession=sqlSessionFactory.openSession();
//ibatis时期方法,这种方式会在JVM常量池中加入很多字符串字面量,消耗内存,返回对象要强制类型转换(测试
//mybatis3.30已经不需要强制类型转换了)
Book book=(Book)sqlSession.selectOne("com.gzl.dao.BookDao.queryById",id);
//新时期方法,参数XxxXxx.class已经可以确定dao接口的全限定名,配合.methodName()调用方法达到了之前
//的效果,而且不必考虑字面量导致的内存消耗和返回类型强制转换的问题.
Book book2=sqlSession.getMapper(BookDao.class).queryById(1000);

MyBatis与Spring进行整合

Spring最关键的功能就是IOC和DI,如果Mybatis基础上与Spring进行整合就可以将Mybatis中最关键的对象SqlSessionFactory用Spring进行IOC(Inversion Of Control),SqlSessionFactory中所需要的dataSource属性(mapper.xml加载位置mapperLocations,实体类别名typeAliasesPackage也可以作为其中的属性)进行DI(Dependency Injection)
上面括号中提到过mapperLocations和typeAliasesPackage也可以作为dataSource的属性进行依赖注入,所以mybatis-config.xml中目前什么都不需要配置,甚至不需要存在.

Spring和Mybatis整合后,当前能干什么

以前后端分离式项目为例,当前dao层(Data Access Objects数据访问层)已经能通过Mybatis帮助下在操作数据库,在dao层基础之上建立service层(业务层),按照现在的进度接下来是建立controller层来实现前端后端的桥梁,当前我们能很方便的操作数据库(mybatis)(写mapper.xml,mapper接口),管理对象(spring)(IOC,DI),但是写controller还是要不耐其烦的写Servlet,现在急需一种框架可以像写普通类一样写controller,方便的映射controller对应的url.
虽然新版本的Servlet3.0标准增加了注释开发,但是更流行的方式是Struts,Struts2框架,目前Struts2应用场景下已经被SpringMVC替代,接下来进行SpringMVC的整合.
