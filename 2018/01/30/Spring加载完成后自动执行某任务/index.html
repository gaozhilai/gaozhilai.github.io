Spring加载完成后自动执行某任务

2018-01-30 • 标签: Spring, 框架

敲黑板划重点

实现ApplicationListener接口,具体实现代码如下:

@Override
   public void onApplicationEvent(ContextRefreshedEvent event)
   {
       if(event.getApplicationContext().getParent() == null)//root application context 没有parent，他就是老大.
       {
           //要执行的逻辑代码
       }

       /*//或者下面这种方式
       if(event.getApplicationContext().getDisplayName().equals("Root WebApplicationContext"))
       {
           //要执行的逻辑代码
       }*/
   }

需求

项目中需要在WebApplication启动后执行一段代码,监听某一端口实现Dicom文件接受服务器的功能.

问题

网上有很多类似上面的实现方法,当WebApplication加载完成后就执行实现了ApplicationListener接口中的逻辑代码,但是我在项目中添加了相关代码后不生效

解决方法

在查看控制台日志的时候发现了一个输出语句

No Spring WebApplicationInitializer types detected on classpath

在classpath中没有Spring WebApplicationInitializer类型被发现.原因是因为在web.xml中我没有配置ContextLoaderListener,这个个监听器负责监听实现ApplicationListener接口的类,从而告诉Spring加载完成后执行指定类的代码.配置过后问题解决.具体配置代码如下: 

<listener>  
  <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>  
</listener>
