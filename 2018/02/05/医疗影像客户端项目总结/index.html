Swing皮肤

Java中的Swing工具包很强大,但是比较明显的缺点就是原生样式不太美观,在解决样式问题的时候发现了Swing的一个Look&Feel开源项目,Darcula皮肤是idea编译器作者开源的项目,就是编译器中著名的黑色皮肤.有人说用java开发的idea编译器把Swing工具包运用到了极致,那么目前这个358kb的jar包就是其中的精髓,Darcula皮肤几乎完全重写了Swing组件的样式,很美观.
项目github地址:
https://github.com/bulenkov/Darcula

GridBagLayout布局方式

要用Java进行客户端开发就怎么也绕不过布局管理器,Java提供了很多布局管理器,最常见的就是大多数组件默认的FlowLayout布局.众多布局管理器中最复杂最强大的就是GridBagLayout,其实说它是Java中最复杂的布局管理器,但是学习成本其实并不高,灵活使用GridbagLayout可以开发出任何想要的布局,按比例适配不同大小的屏幕.

Java Network Launching Protocol

JNLP解决了项目的部署更新和如何被调用的两个问题.
①当实际部署的时候可能要部署的客户机数量很庞大,如果每一个客户机都要部署客户端jar包那么部署人员的压力会很大,每次客户端更新升级代价也会很大.通过JNLP协议解决了这一问题,在服务器端暴露一个动态生成jnlp文件的接口,这样客户机只需要点击对应jnlp协议的链接,JRE就会完成jar包下载更新的动作,jar包只会在第一次下载,不用担心对带宽造成压力.这样更新部署客户端的代价一下子就由很多客户机变成了只需要更新服务器端.

②Java没有提供程序单例运行的机制,也就是说java客户端在被其他系统调用执行时没有办法每次都将参数传递给同一个正在运行Java客户端实例,无论是网上文件锁还是开启端口号都不是好的解决办法,也不能解决参数传递的问题,如果采用了jnlp部署的方式这个问题就迎刃而解了.面对合作方谈接口时也有了很好的规范.

遇到的问题,学到的经验

①面对需求既要合理评估能不能实现,又要有勇于挑战的精神.有很多技术只是名字看起来难,实际并不难.
②开发项目的时候一定要严格分出层次结构,将重复代码进行封装,不然以后维护更新都很痛苦.
③使用JNLP部署项目时,在开发过程中一定要注意异常捕获,日志记录,很多时候报错在jnlp启动时只是表现出无法启动,有限的异常反馈,这时通过异常捕获弹窗,日志定位错误就很有必要了.
④JNLP更新客户端可能会出现不能及时更新的情况,解决方式是在jar包中加入对应版本号,每次更新一个版本.
⑤注意写注释,注意代码格式,保持良好编码风格.
