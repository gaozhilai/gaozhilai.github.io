常见的代码规范提示含义

代码设计风格应该遵循高内聚低耦合,代码格式也应该规范,下面列举出常见的警告.

1)
Type is missing a javadoc commentClass
缺少javadoc注释说明
2)”{“ should be on the previous line
“{“ 应该位于前一行
3)Method is missing a javadoc comment
方法前缺少javadoc注释
4)Expected @ghrows tag for “Exception”
注释中应该有@thows的说明
5)”.” Is proceeded with whitespace
“.” 前面不能有空格
6)”.” Is followed by whitespace
“.” 后面不能有空格
7)”=” is not preceeded with whitespace
“=” 前面缺少空格
8)”=” is not followed with whitespace
“=” 后面缺少空格
9)”}” should be on the same line
“}” 应该放到下一行的前面
10)Unused @param tag for “unused”
没有参数 “unused” ,不需要此注释
11)Variable “unused” missingjavadoc
变量 “unused” 缺少javadoc注释
12)Line longer than 80 characters
行字符长度超过80
13)Line contains a tab character
行中包含制表符
14)Redundant “public” modifier
冗余的 “public” 修饰符
15)Final modifier out of order with the JSL suggestion
final修饰符所在位置不是推荐的顺序
16)Avoid using the “.” formof import
应该避免使用”“引入依赖
17)Redundant import from the same package
从一个包中引入内容
18)Unusedimport-java.util.list
导入的依赖没有被使用
19)Duplicate import to line 13
重复引入了同一内容
20)Import from illegalpackage
引入了非法包的内容
21)”while” construct must use “{}”
“while” 必须使用 “{}”
22)Variable “v1” must be private and have accessor method
变量v1应该是私有的并且提供调用的方法
23)Variable “ABC” must matchpattern “^[a-z][a-zA-Z0-9]$”
变量ABC必须符合命名规则 “^[a-z][a-zA-Z0-9]$” ,例如aBC
24)”(“ is followed by whitespace “)”is proceeded by whitespace
“(“后面不能有空格,”)”前面不能有空格
25)First sentence should end with a period.
注释的语句应该以”.”结束
26)Redundant throws:’NameNotFoundException’ is subclass of ‘NamingException’.
冗余的抛出了一个异常的子类异常
27)Parameter docType should be final.
参数docType应该用final修饰
28)Line has trailing spaces.
有多余的空格
29)Must have at least one statement.
至少要有一个声明
30)’>’ is not followed by whitespace.并且又有 ‘(‘is preceded with whitespace.
“>”后面要有空格,”(“前面不能有空格
31)Got an exception - java.lang.RuntimeException: Unable to get classinformation for @throws tag ‘SystemException’.
不合理的抛出异常
32) File does not end with a new line.
文件没有以新的一行结束
33)Utility classes should not have a public or default constructor.
接口中的内部类不应该有公共或者默认的构造方法
