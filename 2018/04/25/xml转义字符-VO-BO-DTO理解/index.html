VO,BO,DTO理解

POJO(Plain Ordinary Java Object)包括了VO(View Object),DTO(Data Transfer Object),BO(BusinessObject),Entity对象.
DTO用于接收前端参数
VO用于给前端返回结果
BO包含多个VO字段,用于业务层处理需要或者返回结果需要
当一个POJO用于接收参数身份属于DTO,当用这个POJO返回结果的时候,这个POJO就叫做VO
一个POJO应用场景不同所属类型会转变,所谓的VO,BO,DTO是在某个应用场景的角度来定义POJO
